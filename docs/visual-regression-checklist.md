# Zigo Visual Regression Checklist

Use this before demos and APK builds to keep the Instagram-style polish consistent.

## Home Feed

- Feed cards are media-first and mobile-width.
- Story rail uses colorful rings and does not feel empty in preview mode.
- Action rail has like, comment, share and save with clear feedback.
- Captions and comment prompts are visually separated.
- Follow buttons are compact and do not overflow creator names.

## Micro

- Micro fills the vertical viewport.
- Top tabs remain readable over video/gradient media.
- Education reward copy is secondary and does not dominate the reel.
- Bottom creator/caption area does not collide with action rail.
- Overlay bottom nav remains legible.

## Sparks

- Segment progress appears at the top.
- Match-Feed and safety labels stay secondary or screen-reader-only.
- Reply reactions are easy to tap.
- Story media keeps a vertical, mobile-first composition.
- Holding a story pauses both progress and video.

## Profile

- Avatar, stats and action buttons fit one mobile screen.
- Highlights scroll horizontally without showing a scrollbar.
- Profile grid has post/reel/saved badges.
- Empty grid states have clear CTAs.

## Explore

- Search bar sticks correctly below the top app bar.
- Category chips are horizontally scrollable.
- Suggested creator rail appears in no-query discovery mode.
- Masonry discovery tiles use varied sizes.
- Tile labels and video badges remain readable.

## Create

- Media preview uses the right aspect ratio for post vs reel/story.
- Story composer shows an assigned-area selector before publishing.
- Upload/publish progress steps are visible.
- Error messages are understandable.
- Changing media updates filename and preview.

## Student And Parent

- Student league path and streak panel look gamified.
- Duels page clearly says no student DMs.
- Family setup reads as a three-step parent-child flow.
- Store reward cards keep parent approval visible.

## Loading States

- Home loading uses feed-style skeleton.
- Explore loading uses search, creator rail and masonry skeleton.
- Profile loading uses avatar, stats and grid skeleton.
- Micro loading uses full-screen dark video skeleton.

## Empty And Offline States

- Shared empty states look like polished social placeholders, not setup errors.
- Signed-out, no-interest and no-content states provide one clear next action.
- Offline fallback is clean, mobile-width and does not cache stale HTML.
